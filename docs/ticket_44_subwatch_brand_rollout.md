# Ticket 44: SubWatch Brand Rollout

## Brand intent
SubWatch should feel careful, watchful, calm, and premium.

It should not feel aggressive, destructive, finance-coded, or gimmicky.

## Rolled out surfaces
- Android app label and Flutter app title now use `SubWatch`.
- In-app shell title, trust sheet title, and completion-layer brand mentions now use `SubWatch`.
- Launcher icons now use a dark premium base with an amber watch-and-signal mark.
- Android launch background now uses the same icon mark on a warm dark background.
- Post-splash loading state now carries a minimal SubWatch lockup.

## Intentional non-changes
- Internal package/import name `sub_killer` was left unchanged to avoid unnecessary repo-wide churn.
- Platform channel names were left unchanged because they are internal wiring, not user-facing branding.
- No runtime behavior, subscription truth, review semantics, or sync behavior changed.
